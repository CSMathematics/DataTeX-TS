# Contributing

We welcome contributions to DataTex!

## Workflow

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Create a branch** for your feature or fix (`git checkout -b feature/amazing-feature`).
4.  **Make your changes**.
5.  **Test your changes** (run `pnpm tauri dev` to verify).
6.  **Commit and Push** your changes.
7.  **Open a Pull Request** to the `main` branch.

## Code Style

- **TypeScript**: We use `eslint` and `prettier`. Run `pnpm lint` before committing.
- **Rust**: Use `cargo fmt` to format your Rust code.
- **Commits**: Please use descriptive commit messages (e.g., "Fix: Resolve issue with table wizard alignment").
